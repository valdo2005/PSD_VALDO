Preprocessing Data#

Preprocessing Data merupakan proses menyiapkan data mentah agar layak digunakan dalam analisis atau pemodelan dengan cara membersihkan, mentransformasi, dan mengorganisasi data sehingga menjadi lebih konsisten, akurat, dan sesuai kebutuhan metode analisis.

Mendeteksi Outlier pada data Iris#

Kita akan mendeteksi outlier menggunakan beberapa metode. Hal ini dilakukan di Power BI-> Transform Data-> Trasnform-> Run Python Script

1. Deteksi Outlier Menggunakan model ABOD#

ABOD (Angle-Based Outlier Detection) adalah salah satu metode deteksi outlier yang sangat efektif, terutama pada data berdimensi tinggi (memiliki banyak kolom atau fitur). Berbeda dengan metode lain seperti Isolation Forest yang menggunakan jarak, ABOD menggunakan sudut antar titik data untuk mendeteksi outlier.

Secara sederhana, ABOD mengasumsikan bahwa:

  • Untuk sebuah titik data normal (bukan outlier), sudut antara titik tersebut dengan dua titik lainnya di sekitarnya cenderung bervariasi secara luas. Titik normal dikelilingi oleh banyak tetangga yang membentuk berbagai sudut.

  • Untuk sebuah titik data outlier, sudut yang terbentuk antara titik tersebut dan tetangga-tetangganya cenderung kecil atau sempit. Hal ini karena titik outlier berada di “ujung” atau terisolasi dari sebagian besar data lainnya.

ABOD menghitung varians dari sudut-sudut ini. Jika variansnya sangat rendah (sudutnya semua kecil), titik tersebut kemungkinan besar adalah outlier. Kelebihan ABOD adalah kemampuannya mendeteksi outlier tanpa terpengaruh oleh jarak yang bisa menjadi bias pada data berdimensi tinggi.

import pandas as pd
from pyod.models.abod import ABOD

# Power BI secara otomatis menempatkan data dari langkah sebelumnya ke dalam DataFrame bernama 'dataset'
df = dataset.copy()

# Inisialisasi model ABOD
# Ganti nama_kolom_numerik_saja dengan daftar kolom yang ingin Anda analisis
clf = ABOD(n_neighbors=5)
clf.fit(df.select_dtypes(include=['number']))

# Dapatkan prediksi outlier (0 = inlier, 1 = outlier)
outlier_pred = clf.labels_

# Tambahkan kolom 'is_outlier' ke DataFrame
df['is_outlier'] = outlier_pred

# Filter DataFrame untuk hanya menampilkan baris yang terdeteksi sebagai outlier
outliers_only = df[df['is_outlier'] == 1]

# Tampilkan DataFrame yang berisi outlier saja
output = outliers_only

Penjelasan script Python#

  • Impor Pustaka: import pandas as pd untuk manipulasi data dan from pyod.models.abod import ABOD untuk model deteksi outlier.

  • Pemuatan Data: df = dataset.copy(). Power BI secara otomatis memuat tabel yang masuk ke skrip sebagai DataFrame bernama dataset. Skrip ini menyalinnya ke variabel df untuk diproses.

  • Inisialisasi Model: clf = ABOD(n_neighbors=5). Baris ini membuat instance dari model ABOD. n_neighbors adalah parameter penting yang menentukan berapa banyak tetangga terdekat yang digunakan untuk menghitung sudut. Anda bisa menyesuaikan nilainya.

  • Pelatihan Model: clf.fit(df.select_dtypes(include=[‘number’])). Ini adalah bagian yang paling krusial. Alih-alih menyebutkan nama kolom satu per satu, df.select_dtypes(include=[‘number’]) secara otomatis memilih semua kolom yang bertipe data numerik di DataFrame. Model ABOD kemudian dilatih menggunakan kolom-kolom numerik ini untuk menghitung sudut dan mendeteksi anomali.

  • Prediksi dan Penandaan Outlier: outlier_pred = clf.labels_ dan df[‘is_outlier’] = outlier_pred. Setelah model dilatih, ia menghasilkan array (daftar) yang berisi nilai 0 untuk data normal dan 1 untuk data outlier. Skrip kemudian menambahkan array ini sebagai kolom baru bernama is_outlier ke DataFrame asli.

  • Penyaringan Data: outliers_only = df[df[‘is_outlier’] == 1]. Baris ini menyaring DataFrame df dan hanya menyisakan baris-baris di mana kolom is_outlier bernilai 1. Hasilnya adalah DataFrame baru yang hanya berisi data outlier.

  • Output Akhir: output = outliers_only. Terakhir, skrip menetapkan DataFrame outliers_only sebagai output. Power BI akan menampilkan DataFrame ini sebagai tabel baru, yang berisi semua data yang terdeteksi sebagai outlier.

2. Deteksi Outlier menggunakan Model LOF#

LOF adalah metode deteksi outlier yang sangat populer, terutama karena kemampuannya mendeteksi outlier dalam data yang memiliki kepadatan (densitas) berbeda. Secara sederhana, LOF mengukur seberapa terisolasi sebuah titik data dibandingkan dengan kepadatan tetangga-tetangganya.

LOF menghitung “skor outlier” untuk setiap titik data. Semakin tinggi skor LOF, semakin besar kemungkinan titik tersebut adalah outlier. Prosesnya bekerja seperti ini:

  • Menghitung Kepadatan Lokal: Untuk setiap titik data, LOF melihat k tetangga terdekatnya. Kemudian, ia menghitung seberapa padat area di sekitar titik tersebut berdasarkan jarak rata-rata ke tetangga-tetangganya. Area yang padat akan memiliki jarak yang lebih pendek, sedangkan area yang jarang memiliki jarak yang lebih panjang.

  • Membandingkan Kepadatan: LOF membandingkan kepadatan lokal suatu titik data dengan kepadatan rata-rata dari k tetangganya.

    • Jika kepadatan suatu titik mirip dengan tetangganya, skor LOF-nya akan mendekati 1. Ini menandakan titik tersebut adalah normal (bukan outlier).

    • Jika kepadatan suatu titik jauh lebih rendah daripada kepadatan tetangganya, skor LOF-nya akan lebih besar dari 1. Ini menandakan titik tersebut berada di area yang jarang, atau dengan kata lain, outlier.

Keunggulan LOF adalah kemampuannya mendeteksi outlier di dalam klaster data, bukan hanya di area kosong. Ini menjadikannya alat yang sangat andal untuk mendeteksi outlier pada data yang kompleks.

import pandas as pd
from pyod.models.lof import LOF

# Power BI secara otomatis menempatkan data dari langkah sebelumnya ke dalam DataFrame bernama 'dataset'
df = dataset.copy()

# Pilih hanya kolom numerik untuk analisis LOF
df_numeric = df.select_dtypes(include=['number'])

# Inisialisasi model LOF
# n_neighbors=20 adalah contoh, Anda bisa menyesuaikan nilainya
clf = LOF(n_neighbors=20)

# Latih model untuk mendeteksi outlier
clf.fit(df_numeric)

# Dapatkan prediksi outlier (0 = inlier, 1 = outlier)
outlier_pred = clf.labels_

# Tambahkan kolom 'is_outlier' ke DataFrame asli
df['is_outlier'] = outlier_pred

# Filter DataFrame untuk hanya menampilkan baris yang terdeteksi sebagai outlier
outliers_only = df[df['is_outlier'] == 1]

# Tampilkan DataFrame yang berisi outlier saja
output = outliers_only

Penjelasan Skrip#

  • Pemuatan Data: Skrip ini dimulai dengan menyalin DataFrame dataset ke variabel df.

  • Pemilihan Kolom: df.select_dtypes(include=[‘number’]) adalah langkah penting yang memastikan hanya kolom dengan tipe data numerik yang digunakan. Ini akan mencegah kesalahan jika ada kolom teks atau kategorikal di data Anda.

  • Inisialisasi Model: clf = LOF(n_neighbors=20) membuat model LOF. Parameter n_neighbors (jumlah tetangga terdekat) adalah salah satu parameter terpenting yang dapat Anda sesuaikan. Nilai yang umum digunakan adalah 10 hingga 30.

  • Deteksi Outlier: clf.fit(df_numeric) melatih model pada data numerik Anda. Model kemudian menghasilkan label 0 atau 1 untuk setiap baris data, di mana 1 menandakan outlier.

  • Penyaringan Data: Skrip ini menambahkan label outlier ke DataFrame asli, lalu menggunakan df[‘is_outlier’] == 1 untuk menyaring dan menampilkan hanya baris-baris yang terdeteksi sebagai outlier.

  • Hasil Akhir: Variabel output berisi DataFrame yang telah disaring, yang kemudian akan ditampilkan oleh Power BI.

Perbedaan LOF dengan ABOD#

Secara konseptual, LOF juga mendeteksi outlier, tetapi menggunakan pendekatan yang berbeda dari ABOD. LOF mengukur densitas (kepadatan) lokal suatu titik data dibandingkan dengan kepadatan tetangga-tetangganya. Jika kepadatan suatu titik jauh lebih rendah daripada kepadatan tetangganya, titik tersebut dianggap sebagai outlier. LOF sangat efektif untuk mendeteksi outlier pada kelompok data yang berdensitas berbeda.

3. Deteksi Outlier Menggunakan Model IQR#

Metode IQR adalah salah satu pendekatan statistik paling dasar dan umum untuk mendeteksi outlier. Metode ini sangat cocok untuk data yang distribusinya mendekati normal (berbentuk lonceng) karena sangat bergantung pada kuartil.

Cara kerjanya sederhana:

  • Kuartil (Q1 dan Q3): Pertama, data dibagi menjadi empat bagian yang sama.

  • Kuartil Pertama (Q1) adalah nilai yang memisahkan 25% data terendah.

  • Kuartil Ketiga (Q3) adalah nilai yang memisahkan 25% data tertinggi.

  • Jarak Interkuartil (IQR): Ini adalah rentang antara Q1 dan Q3 (IQR=Q3−Q1). IQR mewakili 50% data di tengah.

  • Penentuan Batas Outlier: Outlier didefinisikan sebagai titik data yang berada di luar batas yang ditentukan. Batas ini dihitung berdasarkan IQR:

    • Batas Bawah: Q1−(1.5×IQR)

    • Batas Atas: Q3+(1.5×IQR)

Setiap titik data yang jatuh di luar rentang ini dianggap sebagai outlier. Angka 1.5 ini adalah konvensi statistik yang umum digunakan dan berfungsi sebagai pengali untuk memperlebar batas pencarian outlier.

import pandas as pd

# Power BI secara otomatis menempatkan data dari langkah sebelumnya ke dalam DataFrame bernama 'dataset'
df = dataset.copy()

# Pilih hanya kolom numerik untuk analisis IQR
numeric_cols = df.select_dtypes(include=['number']).columns

# Buat DataFrame kosong untuk menyimpan outlier
outliers = pd.DataFrame()

# Loop melalui setiap kolom numerik untuk mendeteksi outlier
for col in numeric_cols:
    Q1 = df[col].quantile(0.25)
    Q3 = df[col].quantile(0.75)
    IQR = Q3 - Q1

    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR

    # Deteksi outlier
    outlier_rows = df[(df[col] < lower_bound) | (df[col] > upper_bound)]

    # Tambahkan outlier yang terdeteksi ke DataFrame 'outliers'
    outliers = pd.concat([outliers, outlier_rows])

# Hapus duplikasi baris, karena satu baris bisa menjadi outlier di lebih dari satu kolom
outliers = outliers.drop_duplicates().reset_index(drop=True)

# Tampilkan DataFrame yang berisi outlier saja
output = outliers

Penjelasan Skrip#

  • Pemuatan dan Pemilihan Kolom: Skrip dimulai dengan menyalin dataset ke df dan kemudian mengidentifikasi semua kolom numerik. Metode IQR hanya dapat diterapkan pada data numerik.

  • Perhitungan IQR: Skrip akan melakukan perulangan pada setiap kolom numerik untuk menghitung:

    • Q1 (Kuartil Pertama): Nilai yang memisahkan 25% data terendah.

    • Q3 (Kuartil Ketiga): Nilai yang memisahkan 25% data tertinggi.

    • IQR (Interquartile Range): Jarak antara Q3 dan Q1 (Q3 - Q1).

  • Penentuan Batas Outlier: Titik data dianggap outlier jika berada di luar batas bawah dan batas atas yang ditentukan:

    • Batas Bawah: Q1 - 1.5 * IQR

    • Batas Atas: Q3 + 1.5 * IQR

Angka 1.5 adalah konstanta umum yang digunakan dalam metode IQR. Setiap data di luar rentang ini dianggap sebagai outlier.

  • Deteksi dan Penggabungan Outlier:

    • outlier_rows = df[(df[col] < lower_bound) | (df[col] > upper_bound)] berfungsi untuk menyaring baris yang memenuhi kriteria outlier untuk kolom tertentu.

    • pd.concat([outliers, outlier_rows]) menggabungkan outlier dari setiap kolom ke dalam satu DataFrame tunggal.

  • Pembersihan Duplikasi: outliers.drop_duplicates() akan menghapus baris-baris yang muncul lebih dari satu kali. Ini penting karena satu baris bisa menjadi outlier di beberapa kolom.

  • Output Akhir: DataFrame outliers yang sudah bersih dari duplikasi akan menjadi hasil akhir (output) yang ditampilkan oleh Power BI.

4. Deteksi Outlier Menggunakan Metode KNN#

Metode KNN adalah algoritma yang pada dasarnya berbasis pada asumsi bahwa titik data normal berada di dekat titik data normal lainnya, sementara outlier berada jauh dari titik data lainnya.

KNN bekerja dengan menghitung jarak setiap titik data ke k tetangga terdekatnya. k di sini adalah parameter yang kita tentukan (misalnya, k=5). Prosesnya adalah sebagai berikut:

  • Menemukan Tetangga Terdekat: Untuk setiap titik data, algoritma mencari k tetangga terdekatnya di dalam dataset.

  • Menghitung Jarak Rata-rata: Setelah k tetangga ditemukan, algoritma menghitung jarak rata-rata dari titik data tersebut ke k tetangganya.

  • Membuat Keputusan:

    • Jika sebuah titik data berada di area yang padat, jarak rata-ratanya ke k tetangga terdekat akan pendek. Ini menandakan titik tersebut normal.

    • Jika sebuah titik data berada di area yang jarang, jarak rata-ratanya ke k tetangga terdekat akan panjang. Ini menandakan titik tersebut outlier.

Hasil akhirnya adalah skor outlier untuk setiap titik, yang biasanya merupakan jarak rata-rata ke tetangga terdekatnya. Semakin besar skornya, semakin besar kemungkinan titik tersebut adalah outlier.

import pandas as pd
from pyod.models.knn import KNN

# Power BI secara otomatis menempatkan data dari langkah sebelumnya ke dalam DataFrame bernama 'dataset'
df = dataset.copy()

# Pilih hanya kolom numerik untuk analisis KNN
df_numeric = df.select_dtypes(include=['number'])

# Inisialisasi model KNN
# n_neighbors=5 adalah contoh, Anda bisa menyesuaikan nilainya
clf = KNN(n_neighbors=5)

# Latih model untuk mendeteksi outlier
clf.fit(df_numeric)

# Dapatkan prediksi outlier (0 = inlier, 1 = outlier)
outlier_pred = clf.labels_

# Tambahkan kolom 'is_outlier' ke DataFrame asli
df['is_outlier'] = outlier_pred

# Filter DataFrame untuk hanya menampilkan baris yang terdeteksi sebagai outlier
outliers_only = df[df['is_outlier'] == 1]

# Tampilkan DataFrame yang berisi outlier saja
output = outliers_only

Penjelasan Skrip#

  • Pemuatan dan Pemilihan Kolom: Skrip ini dimulai dengan menyalin DataFrame dataset ke df dan kemudian memilih hanya kolom-kolom numerik, yang diperlukan untuk perhitungan jarak oleh model KNN.

  • Inisialisasi Model KNN: clf = KNN(n_neighbors=5) membuat model KNN. Parameter n_neighbors menentukan berapa banyak tetangga terdekat yang harus diperhitungkan untuk menentukan apakah suatu titik adalah outlier atau tidak. Nilai ini sangat penting dan dapat memengaruhi hasil deteksi.

  • Deteksi Outlier: clf.fit(df_numeric) melatih model pada data numerik Anda. KNN akan menghitung jarak rata-rata dari setiap titik data ke k tetangga terdekatnya. Semakin besar jarak ini, semakin terisolasi titik tersebut. Model kemudian akan memberikan label outlier (1) atau normal (0) berdasarkan ambang batas jarak.

  • Penyaringan Data: Skrip menambahkan label outlier ke DataFrame asli, lalu menyaringnya untuk hanya menyertakan baris-baris di mana kolom is_outlier memiliki nilai 1, yaitu yang terdeteksi sebagai outlier.

  • Hasil Akhir: Variabel output berisi DataFrame yang telah disaring, yang kemudian akan ditampilkan oleh Power BI.

Menghapus Data Outlier#

Setelah mendeteksi Outlier menggunakan beberapa metode, kita akan memilih metode yang dianggap lebih unggul dalam mendeteksi outlier. Dari semua metode yang sudah digunakan, metode LOF dianggap lebih unggul dalam mengukur kepadatan relatif, yang membuatnya sangat sensitif terhadap outlier. Oleh karena itu, saya akan menggunakan LOF sebagai metode terbaik dalam skrip ini.

import pandas as pd
from pyod.models.lof import LOF

# Power BI menempatkan data dari langkah sebelumnya ke dalam DataFrame bernama 'dataset'
df = dataset.copy()

# Pilih hanya kolom numerik untuk analisis LOF
df_numeric = df.select_dtypes(include=['number'])

# Inisialisasi model LOF
clf = LOF(n_neighbors=20)

# Latih model untuk mendeteksi outlier
clf.fit(df_numeric)

# Dapatkan prediksi outlier (0 = inlier, 1 = outlier)
outlier_pred = clf.labels_

# Tambahkan kolom 'is_outlier' ke DataFrame asli
df['is_outlier'] = outlier_pred

# Filter DataFrame untuk HANYA MENYIMPAN baris yang bukan outlier
# Data yang memiliki nilai 'is_outlier' == 0 akan dipertahankan
df_clean = df[df['is_outlier'] == 0]

# Tampilkan DataFrame yang sudah bersih (hapus kolom penanda outlier)
output = df_clean.drop('is_outlier', axis=1)

Penjelasan Perubahan Skrip#

  • Baris Kunci yang Berbeda: Perubahan utama terletak pada baris df_clean = df[df[‘is_outlier’] == 0].

  • Fungsi: Skrip sebelumnya menyaring data dengan kondisi == 1 untuk menampilkan outlier. Skrip yang sekarang menggunakan kondisi == 0 untuk memilih data yang bukan outlier (yang sering disebut “inlier”).

  • Hasil Akhir: output kini berisi DataFrame yang telah dibersihkan dari data outlier, siap untuk analisis lebih lanjut di Power BI.
